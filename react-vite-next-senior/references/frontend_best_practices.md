## Default expectations for PRs

- Clear boundaries (UI / feature / domain / data)
- Minimal client JS (especially in Next)
- Loading/empty/error states included
- Accessibility and keyboard flows considered
- Tests cover behavior and failure paths
- Performance impact considered (bundle and re-render surface)
